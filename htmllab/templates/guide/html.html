HTML 详细指南

超文本标记语言 (HyperText Markup Language) 核心概念与实战

1. 标准页面骨架

每个现代 HTML 页面都应包含以下五个核心部分：

<!DOCTYPE html> <!-- 声明文档类型，告诉浏览器这是 HTML5 -->
<html lang="zh-CN"> <!-- 根元素，指定页面语言 -->
<head>
    <!-- 元数据区域：不直接显示在页面上 -->
    <meta charset="UTF-8"> <!-- 字符编码，防止乱码 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- 移动端适配 -->
    <title>页面标题 - 会显示在浏览器标签上</title>
</head>
<body>
    <!-- 内容区域：所有可见内容都在这里 -->
    <h1>你好，世界！</h1>
</body>
</html>

2. 引用外部资源 (CSS & JS)

为了保持代码整洁，我们通常将样式和脚本放在独立的文件中。

引入 CSS 样式表

使用 <link> 标签，通常放在 <head> 区域内：

<head>
    ...
    <!-- 引入本地 CSS 文件 -->
    <link rel="stylesheet" href="style.css">
    <!-- 引入子目录中的 CSS 文件 -->
    <link rel="stylesheet" href="css/layout.css">
</head>

引入 JavaScript 脚本

使用 <script> 标签，建议放在 </body> 标签结束之前：

<body>
    ... 所有的 HTML 内容 ...

    <!-- 引入外部 JS 文件 -->
    <script src="script.js"></script>
    <!-- 引入子目录中的 JS 文件 -->
    <script src="js/app.js"></script>
</body>
💡 为什么要把脚本放在底部？
浏览器在执行 JavaScript 时会停止渲染页面。放在底部可以确保用户先看到页面内容，提高加载速度体验。

3. 核心标签分类

文本布局

  • <h1> ~ <h6>: 标题。<h1> 最重要，通常每个页面只用一个。
  • <p>: 段落，会自动在上下产生间距。
  • <div>: 无意义的块级容器，用于布局。
  • <span>: 无意义的行内容器，用于包裹一小段文字。

多媒体与链接

  • <a href="目标地址">: 超链接。可以使用 target="_blank" 在新窗口打开。
  • <img src="图片地址" alt="替代文本">: 图片。alt 对 SEO 和无障碍非常重要。

列表

<ul> <!-- 无序列表 -->
    <li>列表项 A</li>
    <li>列表项 B</li>
</ul>

4. 表单与交互

用于收集用户信息：

<form action="/submit" method="POST">
    <label for="username">用户名：</label>
    <input type="text" id="username" name="user" placeholder="请输入姓名">
    
    <button type="submit">提交</button>
</form>

5. 语义化布局 (HTML5)

不要全部使用 <div>，尝试使用这些标签来构建页面：

<header>: 页头
<nav>: 导航栏
<main>: 主体内容
<article>: 独立文章
<section>: 页面章节
<aside>: 侧边栏
<footer>: 页脚
